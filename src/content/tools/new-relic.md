## What is New Relic?

New Relic is an observability platform designed to correlate telemetry data across an entire technology stack. It helps users identify the root causes of issues and reduce mean time to resolution (MTTR). The platform aims to resolve problems at scale before they impact business operations.

## Why use it in the API Lifecycle?

* **Application Performance Monitoring (APM)**: Provides end-to-end visibility across applications and services, connecting performance to business outcomes and helping reduce MTTR.
* **Digital Experience Monitoring**: Monitors real-user experiences across mobile and web to identify and address user friction points and user experience flaws using AI.
* **AI and Agentic Monitoring**: Offers control and real-time monitoring of AI models and agent interactions, including behavior and token usage across the AI stack.
* **Infrastructure Monitoring**: Provides unified visibility for cloud-native and Kubernetes environments, linking infrastructure health directly to application performance.
* **Log Management**: Enables querying of log data from various sources in place, optimizing logging strategies and simplifying compliance.
* **Security Vulnerability Management**: Facilitates the prioritization and resolution of security vulnerabilities by using production impact and AI reasoning.

## Best Practices

* Set up New Relic alerts using baseline thresholds to automatically detect anomalies in API request durations.
